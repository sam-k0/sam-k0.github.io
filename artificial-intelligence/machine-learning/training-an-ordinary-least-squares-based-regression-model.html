Training an Ordinary Least Squares-based regression model

In this note, I'm gonna quickly go over how I managed to train a regression model that yielded that got placed in the top 200 for a kaggle challenge!

Firstly, I wanna clarify that the intro sounds a bit over the top and I have to admit that it is. The challenge that I took part in was from 2016 and closed the "official", prize winning submissions 11 years ago.

But nevertheless, I think it might be interesting to cover the approach I used to land a good score for the solution.

Step 0: Looking at the given data

The first (zeroth) step should always be to make yourself known with the data. Take a look at the dataset, the format, the scale, the encoding, whatever.
The background of the given data are restaurants with different information about them, and the challenge is to create a prediction (regression) model to predict the revenue based on the given metadata.

Id,Open Date,City,CityGroup,Type,P1,P2,P3,P4,...,P34,P35,P36,P37,revenue
0,01/22/2011,Niğde,Other,FC,1,4.0,4.0,4.0,1,2,5,4,...,0,0,0,0,0,0,5653753.0

Above is an excerpt of the data given. As you can see, there's one row for Id, which is just a sequentially counting index for each row, then a date, some string, another string, and.. another string. Additionally, there are some obscure P1 to P37 values that nobody knows where they come from. Lastly, there's the "revenue" field, which is gonna be the target value for our prediction model.

So, the first real step should be to create what's known as a correlation matrix.
The correlation matrix tells us how different values correlate (change) in respect to a target value - in this case the revenue.

But before going ahead and generating this matrix, we should first look at the data scales to answer the following questions:

  • How many different Cities are there?
  • How many different CityGroups are there?
  • How many different Types are there?

All of these matter because models can't be trained on just the strings, instead we wanna create numerical mappings to represent these different classes.

Step 1: Normalizing data

So, after loading the data to dataframes, let's normalize the data:

def preprocess_data_compact(train, test):
	mappings = {}
	# City
	train['City'], uniques = pd.factorize(train['City'])
	mappings['City'] = dict(zip(uniques, range(len(uniques))))
	test['City'] = test['City'].map(mappings['City']).fillna(-1).astype(int)
	
	# City Group
	train['City Group'], uniques = pd.factorize(train['City Group'])
	mappings['City Group'] = dict(zip(uniques, range(len(uniques))))
	test['City Group'] = test['City Group'].map(mappings['City Group']).fillna(-1).astype(int)
	
	# Type
	train['Type'], uniques = pd.factorize(train['Type'])
	mappings['Type'] = dict(zip(uniques, range(len(uniques))))
	test['Type'] = test['Type'].map(mappings['Type']).fillna(-1).astype(int)
	
	# --- Handle dates ---
	for df, is_train in [(train, True), (test, False)]:
		df['Open Date'] = pd.to_datetime(df['Open Date'], format='%m/%d/%Y')
		df['OpenYear'] = df['Open Date'].dt.year
		df['OpenMonth'] = df['Open Date'].dt.month
		df['OpenDay'] = df['Open Date'].dt.day
		# Drop Id only from train
		drop_cols = ['Open Date']
		if is_train:
			drop_cols.append('Id')
		df.drop(columns=drop_cols, inplace=True, errors='ignore')
		
		# Convert to numeric just in case
		df[:] = df.apply(pd.to_numeric, errors='ignore')
	# Optional: print mappings for debugging
	for col, mapping in mappings.items():
		print(f"{col} mapping:", mapping)
	
	return train, test

df_train, df_test = pp.preprocess_data_compact(df_train, df_test)
# --- Normalize numeric columns (Z-score scaling)
# (Skip target column 'revenue')
num_cols = df_train.select_dtypes(include=[np.number]).columns.drop('revenue')
mean = df_train[num_cols].mean()
std = df_train[num_cols].std()

df_train[num_cols] = (df_train[num_cols] - mean) / std
df_test[num_cols] = (df_test[num_cols] - mean) / std

Above code does a couple of things and looks more complicated than it is.

The preprocess_data_compact function first creates mappings for our string fields "City", "City Group" and "Type" for the reason explained earlier:

City mapping: {'İstanbul': 0, 'Ankara': 1, 'Diyarbakır': 2, 'Tokat': 3, 'Gaziantep': 4, 'Afyonkarahisar': 5, 'Edirne': 6, 'Kocaeli': 7, 'Bursa': 8, 'İzmir': 9, 'Sakarya': 10, 'Elazığ': 11, 'Kayseri': 12, 'Eskişehir': 13, 'Şanlıurfa': 14, 'Samsun': 15, 'Adana': 16, 'Antalya': 17, 'Kastamonu': 18, 'Uşak': 19, 'Muğla': 20, 'Kırklareli': 21, 'Konya': 22, 'Karabük': 23, 'Tekirdağ': 24, 'Denizli': 25, 'Balıkesir': 26, 'Aydın': 27, 'Amasya': 28, 'Kütahya': 29, 'Bolu': 30, 'Trabzon': 31, 'Isparta': 32, 'Osmaniye': 33} 

City Group mapping: {'Big Cities': 0, 'Other': 1}


Type mapping: {'IL': 0, 'FC': 1, 'DT': 2}

So now we've got integer values representing these strings. Nice.
Next, we handle the date values by first parsing the string representation and then disassembling it as fine-grained as possible, creating "OpenYear", "OpenMonth" and "OpenDay".

The thought behind this is, that maybe there's a strong correlation between what month restaurants open... Maybe more people go to restaurants that opened in Summer months?

Lastly, the mean() and std() functions normalize the values of all numeric columns to a range of -1 to 1. This is called "Standardization" and it handles far outliers much better than just "min-max-scaling" as large outliers don't skew the scale as much.

Anyways now that all the data is prepared and normalized, we can finally generate a correlation matrix:

On the other hand, "P2", "P28" and "P6" have a strong positive correlation with revenue.

Choosing the predictors

There are other attributes that also show strong correlation, and to be honest its up to you what to choose (start by experimenting with the top X strongest ones) but I elected these as the "most influential" attributes to train my regression model on.

So with this set, training the model is simple enough:

import statsmodels.api as sm
# Select predictors and target
X = df_train[['OpenYear', 'City Group', 'City', "P29", "P13", "P6", "P28", "P2"]]
y = df_train['revenue']
X = sm.add_constant(X)
# Fit OLS regression model
model = sm.OLS(y, X).fit()
print(model.summary())

In above code, we define the predictors and target and add a constant, which adds a column containing only 1.0s. This is necessary for the model to calculate the intercept, the mathematical origin of the regression-fit.
Then define the type of regression model, in this case ordinary least squares, and finally fit it.

Printing the model summary shows a whole lot of stats about the model, for example the F-statistic, which represents "the probability, that the correlation between revenue and features is pure coincidence".

This value is 0.00229, so way underneath the 0.05 mark which is great.

Finally, after doing the same things with the test data and exporting the results to a nice csv file, we're done!
